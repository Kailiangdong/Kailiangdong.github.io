第二章

函数真正价值在于我们把它当作一种数据类型来使用。

如果一个函数中使用了var，那么这个变量就被视作局部变量，只存在于这个函数上下文中， 反之没有使用var，就将视作一个全局变量，如果脚本中已经有一个与之同名的全局变量，那么这个函数就会改变那个全局变量的值。

第三章

DOM document object model
JavaScript 语言中对象可以分为三个类型， 用户定义对象，内建对象，宿主对象。
最基础的对象就是windows对象，对应着浏览器本身，它的属性和方法一般称为BOM。
html文档就是一个节点树，node，有三种，
元素节点（element node）：就是 body,p,ul。
文本节点：夹在元素之间的文本。
属性节点：就是title，class这些

继承是css技术中一项强大的功能，节点树上的各个元素将继承父原色的样式属性。
class属性 , .special{};
id属性， #purchasees{};
获取元素 getElementById
getElementByTagName 这个方法返回一个对象数组，比如说getElementByTagName（”li”)[i],如果你想知道某份文档里总共多少个元素节点，getElementByTagName（“*”）.length.
getElementByClassName是一个新增的方法，也是返回一个对象数组，一般需要判别浏览器能不能支持这个方法，

每个节点都是一个对象

getAttribute是一个函数，它只有一个参数,它不属于document对象，通过元素节点调用，所以有：document.getElementsByTagName(“p”) paras[i].getAttribute(“title”)

setattribute实际上完成了两项操作，先创建这个属性，然后设置它的值
setattribute做出的修改不会反映在文档本身的源码中，这种表里不一的现象源自dom的工作模式，先加载静态内容，再动态刷新，动态刷新不影响文档的静态内容

childNodes属性

一棵节点树上，childNodes 属性可以用来获取任何一个元素的所有子元素

但是childNodes返回不仅仅是元素节点，每一个东西都是节点，空格和换行符也是
每个节点都有nodeType ,元素节点是 1 ， 属性节点是 2 ， 文本节点是 3

nodeValue

改本文本的值就使用nodeValue
description.childNodes[0].nodeValue 得到文本 “Choose an image”

firstChild 和lastChild属性

firstChild相当于 childNodes[0]
lastChild相当于 childNodes[node.childNodes.length - 1]
用nodeValue刷新文字

第五章

document对象是windows对象的一个属性
平稳退化，就是说及时用户屏蔽了javascript，你的网页也能很好的展示。
javascript用windows对象的open()方法来创建新的浏览器窗口
window.open(url,name,features)
第一个参数是新窗口打开url的地址
第二个是新窗口的名字
第三个是新窗口属性 各种长宽高
JavaScript伪协议通过一个链接来调用JavaScript函数

这种做法不是很好
另一种方式是内嵌的事件处理函数，就是之前放在里面的onclick()
平稳退化有啥用，不仅仅是用户，有的搜索机器人也会看，你不平稳退化，搜索排名就不行。

渐进增强：
就是用一些额外的信息层去包裹原始数据，就好比本身html是骨架，css加上之后恩挺漂亮的。

向后兼容
对象检测，一般用一个if语句先去检测，是否满足某个dom条件

性能考虑

尽量少访问dom和尽量减少标记：把搜索结果保存在一个全局变量里，或者把一组原色直接以参数的形式传递给函数
合并脚本：脚本在标记中的位置对页面的初次加载时间也有很大的影响，一般来说，根据http规范，浏览器每次从同一个域名中最多只能同时下载两个文件。
合理方法， 把script放到body后买你， window的onload对象依然可以对文档进行各种操作
压缩脚本，空格和注释，统统删除。我应该有两个脚本，一个是工作副本，修改代码增加注释，一个是精简副本，放在站点上，在精简副本文件名上加上min字样。
